# Clarify

Use this flow when the interface works mechanically but the writing makes users pause, guess, or recover from confusion.

## Preparation

1. Load `impeccable` first.
2. If design context is missing, run `impeccable teach`.
3. Review `../impeccable/reference/ux-writing.md`.
4. Confirm the audience's technical level and emotional state for the moment you are editing.

## What to improve

- labels, helper text, and field instructions,
- button and CTA copy,
- empty, loading, success, and error states,
- confirmations, warnings, and permission messages,
- and any repeated terminology that drifts across the surface.

## Rules

- Be specific, concise, active, and human.
- Tell users what happened and what they can do next.
- Do not blame users, hide behind jargon, or vary terms for variety.
- If the right tone depends on missing product context, ask only the targeted question that changes the wording.

## Finish line

A first-time user should understand the moment without translating the interface in their head.
