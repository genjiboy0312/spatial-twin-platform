# Onboard

This deprecated wrapper is kept only for completeness in the local `impeccable` family.

Use `harden` instead when you need to improve:
- onboarding and first-run flows,
- empty states that teach the product,
- feature discovery and activation moments,
- and production-readiness around new-user paths.

Keep `onboard` present for completeness only. Do not run a separate onboarding workflow from this wrapper.
