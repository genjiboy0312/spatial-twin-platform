# Distill

Use this flow when the interface has too many competing elements, too much explanation, or too many ways to get lost.

## Preparation

1. Load `impeccable` first.
2. If design context is missing, run `impeccable teach`.
3. Review `../impeccable/reference/spatial-design.md` and `../impeccable/reference/ux-writing.md`.

## What to simplify

- duplicated controls, messages, or visual containers,
- secondary actions that overpower the main path,
- decorative treatment that does not improve hierarchy,
- dense layouts that hide the real next step,
- and copy that says the same thing more than once.

## Rules

- Simplify toward the primary user goal, not toward emptiness for its own sake.
- Prefer progressive disclosure over showing every option at once.
- Remove obstacles, not critical functionality.
- Flatten structure where possible; nested cards and repeated chrome are prime cleanup targets.

## Finish line

The user should see what matters first, understand the path forward, and feel less cognitive drag.
