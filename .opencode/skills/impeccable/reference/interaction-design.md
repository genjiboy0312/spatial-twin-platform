# Interaction design reference

Use this reference when the task involves forms, focus states, affordances, or action feedback.

## Core rules

- Make the primary action obvious.
- Give every interactive element default, hover, focus, active, disabled, and loading states where relevant.
- Use progressive disclosure when complexity is real.
- Make empty, success, and error states teach the interface.

## Good interaction patterns

- Inline validation with useful recovery guidance.
- Clear focus visibility for keyboard users.
- Fast-feeling optimistic interactions when failure recovery is well defined.
- Contextual secondary actions instead of overwhelming top-level controls.

## Anti-slop watchouts

- Icon-only controls without labels where recognition suffers.
- Generic skeletons that never resolve the real empty or error story.
- Every button styled as primary.
