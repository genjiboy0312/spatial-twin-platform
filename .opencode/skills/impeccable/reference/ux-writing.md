# UX writing reference

Use this reference when labels, helper text, error states, or empty states feel generic or over-explained.

## Core rules

- Make every line earn its place.
- Prefer specific verbs and nouns over vague reassurance.
- Keep labels and calls to action outcome-oriented.
- Explain what happened, why it matters, and what to do next.

## Good copy habits

- Buttons describe the action.
- Errors preserve user confidence while still being specific.
- Empty states teach the next step.
- Repeated headings and intro copy are cut aggressively.

## Anti-slop watchouts

- Generic AI filler copy.
- Repeating what the user can already see.
- Overly clever microcopy that hides the actual action.
