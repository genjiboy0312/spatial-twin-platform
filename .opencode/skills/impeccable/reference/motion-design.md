# Motion design reference

Use this reference when the interface needs animation, transition polish, or state-change clarity.

## Core rules

- Motion must explain change, not distract from weak hierarchy.
- Prefer transform and opacity over layout-thrashing animation.
- Use natural ease-out curves, not bounce or elastic novelty.
- Respect reduced motion preferences.

## Good motion moments

- Enter and exit transitions for panels, modals, drawers, and route changes.
- Loading transitions that reassure the user without stalling them.
- Staggered reveals for a deliberate first impression.

## Anti-slop watchouts

- Motion added to every element.
- Long ornamental animations blocking usability.
- Bounce easing used as a shortcut for personality.
