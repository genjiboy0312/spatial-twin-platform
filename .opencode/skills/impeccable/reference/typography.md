# Typography reference

Use this reference when the interface needs stronger hierarchy, more deliberate font choices, or better reading comfort.

## Core rules

- Use a small set of sizes with meaningful contrast between levels.
- Use fixed `rem` scales for dense app UI; use fluid heading scales for marketing or editorial surfaces.
- Keep body width near 65-75 characters.
- Loosen line height slightly for light text on dark surfaces.

## Font selection discipline

- Pick fonts from the brand voice, not from model reflexes.
- Reject default-safe choices and near-default replacements.
- Pair a distinctive display voice with a calm, readable body voice when the interface can support it.
- Keep total font families tight and justified.

## What good typography changes

- Faster scanning.
- Stronger emphasis without extra decoration.
- Better emotional alignment with the product.

## Anti-slop watchouts

- Flat hierarchy with many barely different sizes.
- One generic sans used everywhere.
- Long paragraphs without width control.
- Uppercase copy used as body text.
