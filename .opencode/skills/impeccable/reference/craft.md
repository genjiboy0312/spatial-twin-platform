# Craft flow

Use this flow when `impeccable` is invoked with `craft`.

## Flow

1. Confirm design context first. If it does not exist, run `impeccable teach`.
2. Choose the primary implementation pack:
   - `frontend-web` for browser UI
   - `mobile-app` for native or cross-platform mobile UI
3. Load the reference files that match the brief: always typography and spatial design, then add color, interaction, motion, responsive, or UX writing as needed.
4. Build the feature in layers: structure, hierarchy, interaction, edge states, responsiveness, then polish.
5. Run `audit` or `critique` before the final refinement pass.
6. Finish with `polish`, and add `typeset`, `colorize`, or `adapt` if the audit clearly points there.

## Quality bar

- The result should feel intentional in its primary state and its edge states.
- The result should survive the AI slop test.
- The result should look consistent with the domain pack that implemented it.
